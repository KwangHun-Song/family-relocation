# 01. 집 정하기 — 작업계획

## 목표

임장, 후보 리스트업, 비교 분석을 통해 계약할 집을 최종 결정한다.

---

## 하위 단계

### 1단계: 임장 (현장 방문)
- 후보 지역/단지 선정
- 현장 방문 및 메모 작성
- 완료 기준: 주요 후보 단지 3개 이상 방문 완료

### 2단계: 리스트화 (데이터 정리)
- 방문한 집 정보를 비교표로 정리
- 긍정/부정 요소 체계적 기록
- 완료 기준: 비교표 완성, 후보 2~3개로 압축

### 3단계: 결정
- 가족 회의 후 최종 후보 선정
- 호가 협의 및 매물 확보
- 완료 기준: 계약할 집 확정, 계약 일정 잡힘

---

## 데이터 소스

| 구글 시트 탭 | 내용 | 용도 |
|-------------|------|------|
| `임장리스트` | 방문 예정/완료 매물 목록 | 진행사항·현재상황 업데이트 |
| `임장일기` | 방문 후 상세 메모 | 자료/ 폴더 문서 작성 |
| `집결정 긍정/부정` | 후보별 장단점 비교 | 비교 분석 자료 |
| `내집마련일기` | 전반적인 집 구매 여정 기록 | 진행사항 참고 |

---

## 데이터 동기화 방법

```bash
# 임장 리스트 읽기
gws sheets +read \
  --spreadsheet 14Lz-m4gO_a498EjNui1BdJmjTCo5ZVQeERLhoUjvmcQ \
  --range "임장리스트!A1:Z100"

# 임장 일기 읽기
gws sheets +read \
  --spreadsheet 14Lz-m4gO_a498EjNui1BdJmjTCo5ZVQeERLhoUjvmcQ \
  --range "임장일기!A1:Z100"

# 집결정 긍정/부정 읽기
gws sheets +read \
  --spreadsheet 14Lz-m4gO_a498EjNui1BdJmjTCo5ZVQeERLhoUjvmcQ \
  --range "집결정 긍정/부정!A1:Z100"
```

동기화 후:
1. `임장리스트` 데이터 → `자료/임장-매물-목록.md`에 정리
2. `임장일기` 데이터 → `자료/YYYY-MM-DD-임장메모.md`로 날짜별 저장
3. `집결정 긍정/부정` → `자료/후보-비교표.md`에 정리
4. `현재상황.md`의 "후보 현황" 섹션 업데이트
5. `진행사항.md`에 동기화 날짜 기록


## 주요 고려사항

- **토허제(토지거래허가구역)**: 해당 지역 여부 반드시 확인
- **실거주 의무**: 대출 조건에 따른 실거주 요건 확인
- **자금 계획과 연동**: 최종 결정 전 03-대출받기 카테고리와 예산 맞춤 필수
- **시세 변동**: 결정 시점의 실거래가와 호가 차이 추적
