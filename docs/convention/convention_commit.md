# 커밋 컨벤션

> 참고: 01-interview 프로젝트의 commit_convention.md

---

## 커밋 메시지 템플릿

```
[어미] 커밋 내용 요약
- 커밋 세부내용 1
- 커밋 세부내용 2
```

### 어미 종류

| 어미 | 사용 시점 |
|------|----------|
| `feat:` | 새로운 기능 추가 |
| `fix:` | 버그 수정 |
| `docs:` | 문서 추가 또는 수정 |
| `refactor:` | 기능 변경 없는 코드 개선 |
| `test:` | 테스트 추가 또는 수정 |
| `chore:` | 빌드, 설정, 의존성 등 기타 작업 |
| `perf:` | 성능 개선 |
| `ci:` | CI/CD 설정 변경 |

## 규칙

- 첫 줄은 `[어미] 요약` 형식으로 작성한다
- 빈 줄 없이 바로 세부내용을 `-` 목록으로 작성한다
- **항상 한글로 작성한다**
- 세부내용은 구체적인 변경 사항을 나열한다
- 한글로 작성한다
- `git add .` 등 전체 파일 추가를 사용하지 않고, 작업한 파일만 개별 지정하여 커밋한다
- 작업 중 생성된 파일 중 무시해야 할 파일이 있으면 `.gitignore`에 해당 패턴을 추가하는 작업을 병행한다
- 매번 커밋하지 않고, 의미 있는 작업 단위가 완료되었을 때 커밋한다
