# 문서화 지침

> 각 문서의 작성 방법, 데이터 동기화 절차, 자료 관리 기준을 정의합니다.

---

## 1. 작업계획.md 작성 지침

작업계획은 해당 카테고리의 **전략 문서**입니다. 자주 수정하지 않으며, 방향이 바뀔 때만 업데이트합니다.

### 포함해야 할 항목

```markdown
# [카테고리명] 작업계획

## 목표
이 단계에서 달성해야 할 것 (한 문장으로)

## 하위 단계
단계별 작업 목록과 각 단계의 완료 기준

## 데이터 소스
- 구글 시트 탭 이름과 용도
- 외부 참고 자료 링크

## 데이터 동기화 방법
gws 명령어로 데이터를 가져오는 방법

## 문서 구조
이 카테고리 폴더 안 각 파일의 역할

## 주요 고려사항
의사결정 시 반드시 확인해야 할 제약 조건, 리스크
```

---

## 2. 진행사항.md 작성 지침

진행사항은 **타임라인 로그**입니다. 완료된 일을 날짜순으로 기록합니다.

### 형식

```markdown
## YYYY-MM-DD

- [완료] 수행한 작업 내용
- [완료] 확인된 사실 또는 결정사항
- [메모] 다음 단계에 영향을 주는 정보
```

### 작성 원칙
- 과거형으로 작성 ("방문했다", "확인했다")
- 날짜별로 구분, 최신 항목이 **위**에 오도록
- 링크나 파일은 `자료/` 폴더에 저장하고 여기서 참조

---

## 3. 현재상황.md 작성 지침

현재상황은 **스냅샷 문서**입니다. "지금 당장 이 상황이야"를 설명합니다.

### 형식

```markdown
# 현재 상황

> 최종 업데이트: YYYY-MM-DD

## 한 줄 요약
지금 이 카테고리가 어느 단계에 있는지

## 상세 현황
현재 진행 중인 일, 대기 중인 일, 막힌 부분

## 다음 액션
당장 해야 할 가장 중요한 일 1~3가지

## 주요 수치 / 정보
날짜, 금액, 연락처 등 핵심 데이터
```

### 작성 원칙
- 현재형으로 작성 ("진행 중이다", "기다리고 있다")
- 업데이트할 때마다 날짜를 갱신
- 오래된 정보는 진행사항.md로 이동 후 삭제

---

## 4. 투두리스트.md 작성 지침

투두리스트는 **실행 가능한 다음 할 일** 목록입니다.

### 형식

```markdown
# 투두리스트

## 🔴 긴급 (이번 주 안에)
- [ ] 구체적인 행동 (담당자) — 마감일

## 🟡 중요 (이번 달 안에)
- [ ] 구체적인 행동

## 🟢 예정 (다음 단계)
- [ ] 구체적인 행동

## ✅ 완료
- [x] 완료된 항목 — 완료일
```

### 작성 원칙
- 행동 동사로 시작 ("확인하다", "연락하다", "제출하다")
- 모호한 표현 금지 ("알아보기" → "은행에 전화해서 한도 확인하기")
- 완료된 항목은 삭제하지 말고 ✅ 완료 섹션으로 이동

## 5. 자료/ 폴더 관리 지침

### 저장 대상
- 임장 메모, 집 비교표
- 계약서 스캔본 링크 (민감 정보는 링크만)
- 대출 상품 비교 데이터
- 이사업체 견적서

### 파일 명명
```
YYYY-MM-DD-내용-설명.md
예: 2026-04-08-아파트A-임장메모.md
    2026-04-10-은행별-대출금리-비교.md
```

### 민감 정보 처리
- 계좌번호, 주민번호 등 민감 정보는 파일에 직접 기재하지 않음
- 구글 드라이브 링크로 대체하거나 별도 보안 저장소 사용

## 6. 구글 시트 데이터 동기화

이사-회의록 시트(ID: `14Lz-m4gO_a498EjNui1BdJmjTCo5ZVQeERLhoUjvmcQ`)에서 데이터를 가져오는 방법입니다.

### 기본 명령어

```bash
# 특정 탭의 데이터 읽기
gws sheets +read \
  --spreadsheet 14Lz-m4gO_a498EjNui1BdJmjTCo5ZVQeERLhoUjvmcQ \
  --range "탭이름!A1:Z100"
```

### 탭별 데이터 소스 매핑

| 카테고리 | 관련 시트 탭 |
|----------|-------------|
| 01-집-정하기 | `임장리스트`, `임장일기`, `집결정 긍정/부정`, `내집마련일기` |
| 02-계약하기 | `계약준비`, `2026.02.03`, `2026.1월 가정회의`, 날짜별 회의록 |
| 03-대출받기 | `대출정리`, `2026.02 금리`, `부대비용 정리`, `차용증` |
| 04-이사하기 | `이사준비`, `위험` |

### 동기화 절차

1. `gws sheets +read`로 해당 탭 데이터 읽기
2. 내용을 분석해 해당 카테고리 문서에 반영
3. `진행사항.md`에 동기화 날짜와 변경 내용 기록
4. `현재상황.md`의 수치 데이터 업데이트

### 자동화 계획 (웹서비스 연동 시)
- 구글 Sheets API를 통해 실시간 데이터 fetch
- 변경사항 감지 시 자동으로 해당 문서 업데이트
- 웹 대시보드에서 최신 데이터 시각화

## 7. 웹서비스 문서 렌더링 규칙

이 마크다운 문서들은 웹서비스에서 다음과 같이 렌더링될 예정입니다.

- `#` 제목 → 페이지 헤더
- 표 → 인터랙티브 테이블
- `- [ ]` 체크박스 → 클릭 가능한 체크박스
- `자료/` 폴더 → 해당 페이지의 첨부파일 섹션
- 코드 블록 → 실행 가능한 명령어 (개발자 모드)

웹서비스 개발 시 이 규칙에 맞춰 컴포넌트를 설계합니다.
