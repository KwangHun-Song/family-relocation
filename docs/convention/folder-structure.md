## 파일 명명 규칙

### 카테고리 폴더
- 형식: `숫자두자리-이름` (예: `01-집-정하기`)
- 번호는 진행 순서를 반영
- 한글 이름, 공백은 하이픈(`-`)으로 대체

### 문서 파일
- 핵심 문서 4종은 고정 이름 사용: `작업계획.md`, `진행사항.md`, `현재상황.md`, `투두리스트.md`
- `자료/` 폴더 안 파일은 `YYYY-MM-DD-설명.md` 형식 권장 (예: `2026-04-08-아파트-비교표.md`)

### docs/convention
- 지침 문서만 저장
- 파일명은 영문 소문자 + 하이픈

## 웹서비스 연동 계획

이 폴더 구조는 웹서비스로 렌더링될 예정입니다.

- 각 마크다운 파일이 웹 페이지로 변환됨
- 카테고리 폴더 = 웹의 섹션/라우트
- `자료/` 폴더 = 해당 섹션의 첨부 자료 페이지
- 구글 시트 데이터를 API로 가져와 실시간 업데이트 가능하게 구성 예정
