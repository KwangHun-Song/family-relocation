# 문서 작성 스타일 가이드

> 참고: 01-interview 프로젝트의 convention_writing_style.md를 이 프로젝트에 맞게 적용

---

## 언어

- **기본 언어: 한국어**
- 코드, 명령어, 환경변수, 기술 용어는 영어 그대로 사용
- 날짜: `YYYY-MM-DD` 형식 (예: `2026-05-22`)

## 헤더 구조

```
# (H1): 문서 제목 — 문서당 1개
## (H2): 주요 섹션
### (H3): 세부 항목
```

## 상태 표시

| 상태 | 표기 |
|------|------|
| 완료 | `✅` 또는 `[x]` |
| 진행 중 / 예정 | `⏳` 또는 `[ ]` |
| 긴급 | `🔴` |
| 중요 | `🟡` |
| 일반 | `🟢` |
| 이사 | `🚚` |
| 정착 | `🏠` |

## 날짜 표기

- 항상 절대 날짜 사용 (상대 날짜 금지)
- 올바름: `2026-05-22`
- 잘못됨: `다음 주`, `이번 달`

## 테이블 형식

- 현황 문서: 테이블로 정리
- 투두리스트: `- [ ]` / `- [x]` 체크박스 형식
- 비교 자료: 테이블 형식

## 문서 상단 메타정보

```markdown
> 최종 업데이트: YYYY-MM-DD
```

## 코드 블록

- 명령어: `` `backtick` `` 또는 ```bash 블록
- 환경 변수: `` `SCREAMING_SNAKE_CASE` ``
