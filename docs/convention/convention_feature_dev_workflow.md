# 피처 개발 워크플로우

> 참고: 01-interview 프로젝트의 convention_feature_dev_workflow.md를 이 프로젝트에 맞게 적용

---

## 개요

새로운 기능을 개발할 때 아래 단계를 순서대로 진행한다.

---

## 피처 폴더 구조

```
docs/feature/{feature-name}/
├── concept.md          # 개발 컨셉 (왜 만드는가, 무엇을 만드는가)
├── design.md           # 기술 설계 (어떻게 만드는가)
├── implementation_plan.md  # 구현 계획 (누가, 어떤 순서로)
├── trd.md              # 기술 참조 문서 (완성 후 작성)
└── review/             # 리뷰 기록
```

---

## 단계별 진행

### 1단계: 컨셉 문서 작성 (`concept.md`)

- 왜 이 기능이 필요한가 (배경/동기)
- 무엇을 만드는가 (핵심 기능 목록)
- 누가 사용하는가 (대상 사용자)
- 무엇을 만들지 않는가 (범위 제외)

### 2단계: 설계 문서 작성 (`design.md`)

- 기술 스택 선택 및 이유
- 시스템 구조 (폴더 구조, 데이터 흐름)
- 핵심 구현 방식
- 환경 변수 및 배포 설정

### 3단계: 구현 계획 작성 (`implementation_plan.md`)

- 태스크를 병렬/순차 단계로 분리
- 각 태스크별 담당 에이전트 지정
- 완료 기준(Definition of Done) 명시

### 4단계: 구현

- 구현 계획대로 에이전트 실행
- 태스크 완료 시 체크

### 5단계: 검증

- 빌드 통과 여부
- 기능 동작 확인
- Vercel 배포 확인

### 6단계: TRD 작성 (`trd.md`)

- 완성 후 실제 구현 내용을 기록
- 트러블슈팅 기록
